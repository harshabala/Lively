## Typography - Native Apple System

- **Display + body:** Apple system font (`.system`, SF Pro on macOS)
- **Mono:** Apple monospaced system font (`.monospaced`, SF Mono) only for fixed-width values such as timers, counters, file metadata, and debugging details

Do not mix decorative fonts inside the app. Lively should feel like it belongs on macOS. Use weight, size, opacity, spacing, and material depth for hierarchy instead of switching typefaces.

Recommended SwiftUI usage:

```swift
Text("Lively")
    .font(.system(size: 18, weight: .semibold))

Text("00:42")
    .font(.system(size: 12, weight: .medium, design: .monospaced))
```

### Type Scale

| Role | SwiftUI guidance | Use |
|---|---|---|
| App title | `.system(size: 18, weight: .semibold)` | Menu/settings title |
| Section | `.system(size: 13, weight: .semibold)` | Section labels |
| Body | `.system(size: 13, weight: .regular)` | Default controls |
| Caption | `.system(size: 11, weight: .regular)` | Metadata, helper text |
| Mono | `.system(size: 12, weight: .medium, design: .monospaced)` | Durations, file details, stable numbers |

## Gradients

Use gradients sparingly. The app should feel glassy and alive, not like a landing page.

### Background Gradient

Light:

```css
--gradient-bg: linear-gradient(135deg, oklch(0.985 0.012 195) 0%, oklch(0.955 0.017 210) 100%);
```

Dark:

```css
--gradient-bg: linear-gradient(135deg, oklch(0.120 0.018 210) 0%, oklch(0.150 0.023 225) 100%);
```

Use for large settings-window backgrounds or preview surfaces only when the gradient is barely perceptible.

### Accent Gradient

Light:

```css
--gradient-accent: linear-gradient(to right, oklch(0.470 0.130 190) 0%, oklch(0.490 0.140 160) 100%);
```

Dark:

```css
--gradient-accent: linear-gradient(to right, oklch(0.760 0.140 190) 0%, oklch(0.780 0.150 160) 100%);
```

Use for small accents: progress fills, selected-state glints, or a restrained primary action. Do not put body text directly on the accent gradient.

## Tone And Voice

### Words to use

Use measured, plain language. Explain what Lively is doing in terms of the user’s desktop, displays, Spaces, and selected videos. Keep labels short and specific: "Set video", "Preview", "Muted", "Fill", "Fit", "Launch at login".

### Words to avoid

Avoid urgency, hype, exclamation marks, and marketing-style claims. Do not call features magical, revolutionary, powerful, or seamless. The app should sound like a good Mac utility, not a campaign.

### Voice example

> Aurora Loop is assigned to this Space. It will stay muted and fill the display.

## Usage Dos And Don'ts

**Do:**

- Prefer macOS materials, subtle borders, and restrained shadows over saturated panels.
- Keep the dominant palette cool and quiet; use Mist Reef teal for action, focus, and selection.
- Test light and dark appearances side by side.
- Use SF Pro everywhere, with SF Mono only for fixed-width values.
- Use the accent gradient only as a small visual reward.

**Don't:**

- Mix unrelated fonts in the app.
- Use purple/blue gradients as broad backgrounds.
- Turn every surface into a card.
- Hardcode colors in feature views once a SwiftUI brand token layer exists.
- Let decorative motion compete with the user’s actual wallpaper video.

## SwiftUI Mapping Notes

For the redesign, create a small native token layer instead of scattering color literals:

```swift

    static let primary = Color(...)
    static let background = Color(...)
    static let card = Color(...)

    static let mutedText = Color(...)
```
